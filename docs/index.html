Clean commit history and structured code reviews for Bitbucket

Replace Bitbucket’s verbose "Merged in…" messages with your PR title for a readable git history—and inject conventional comment labels directly into PR review editors for consistent, structured feedback.

Add to Chrome Add to Firefox

What you get

Bitbucket’s default merge messages include unnecessary noise like “Merged in <branch>” and “(pull request #X)”. This extension replaces them with the PR title, making your git history cleaner and easier to scan. It also brings conventional comment labels into every PR review editor—no copy-pasting prefixes by hand.

Message formats

Choose default Bitbucket text, PR title only, or title plus commit details—same options as in the app, stored in extension storage.

Per merge strategy

Toggle the formatter independently for merge, squash, and rebase so each workflow behaves the way your team expects.

Conventional comments

Inject label buttons below every PR comment box—click suggestion, issue, praise and more to prepend a structured prefix instantly.

Screenshots

Settings live in the popup and side panel; the merge dialog shows Bitbucket’s UI with your formatted commit message when a replacement applies. Conventional comment labels appear directly below every PR comment editor.
